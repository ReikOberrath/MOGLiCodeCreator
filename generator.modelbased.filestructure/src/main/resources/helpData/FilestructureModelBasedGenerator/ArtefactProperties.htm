The FilestructureModelBasedGenerator plugin (since 1.2.0) defines the following attributes which you can set for your purposes in the artefact.properties file:

1.    TargetDir                                      Folder in your file system where to write the file structure of the given artifact. If not set, the applicationRootDir is used.

2.    RootName                                  Top folder of the artifact’s file structure. If not set,  the artifact name is used as default.

3.    CreateNew                                 If “false” already existing files in the target directory (defined by the ‘targetDir’ property) will not be overwritten. If not set, default ‘true’ is used.

4.    NameOfValidModel                If equal to the name of the model provided by the ModelProvider the artifact is generated. If not set, the artifact is generated for all models. Should be defined before ModelMetaInfo references are used.

5.    OutputEncodingFormat         Encoding applied when replacing placeholders within text files. If not set, ‘UTF-8’ Is used as default.

6.    Exclude                                       Comma separated list of files or directories that do not belong to the artifact and that are ignored when copying the file structure. The artifact.properties file is automatically excluded.

7.    ReplaceIn                                   Defines a replacement of a placeholder within a text file. Placeholders in the defined file are replaced by the text defined here. This text cannot contain line breaks.

8.    RenameFile                                               Defines a file renaming that causes a file from the artifact’s input directory to have a different name in the artifact’s output and target directory.

 

ModelMetaInfo references

When defining the properties listed above, the property values may reference a MetaInfo element of the model (provided by the ModelProvider plugin). A ModelMetaInfo reference is a placeholder which is replaced by a value read from the model file.  Only references to MetaInfo elements in the MODEL hierarchy level are possible.

For example:

@RootName ${ModelMetaInfo=projectName}

When parsing this line, the model’s list of MetaInfo elements is searched for the MetaInfo ‘projectName’. The value of this MetaInfo is used as value for the ‘RootName’ property.
