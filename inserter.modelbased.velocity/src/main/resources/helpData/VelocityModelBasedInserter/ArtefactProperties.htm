The VelocityModelBasedInserter plugin defines the following attributes which you can set for your purposes in a template header:

1.    TargetDir                                      Folder in your file system where to find the target file.

2.    TargetFileName                        Name of file to insert the generated content.

3.    CreateNew                                 If “true” the target file is overwritten if existing. Default is false. The VelocityModelBasedInserter can create a new target file. This means, it can be used as a model based generator.

4.    NameOfValidModel                Determines that the target file is only modified if the name of the model provided by the StandardModelProvider is equal to the model name defined here. If not provided, the template is applied to all models.

5.   OutputEncodingFormat         Encoding applied when generating the artefact’s output file(s). Default is UTF-8. Since 1.1.0

6.   ReplaceStart                              Defines a single line text after which the lines will be replaced by the generated content. This attributes requires the additional attribute “ReplaceEnd”.

7.   ReplaceEnd                                Defines a single line text before which the lines will be replaced by the generated content. This attributes requires the additional attribute “ReplaceStart”.

8.   InsertBelow                                               Defines a single line text after which the generated content is added to the existing file content.

9.   InsertAbove                                              Defines a single line text before which the generated content is added to the existing file content.

10. SkipGeneration                        If “true”, code supposed to be created is not generated. Default is false. Used to suppress generation of files when the VelocityModelBasedInserter is used as a model based generator (see CreateNew).
